## 本任务交付的观测能力（非行为修复）

1. **Render / overlay decision 观测**（读 production 状态）
   - `ScrollingSpace::render_observation()`（`#[cfg(test)]`）
   - 字段：maximize FSM（Idle/Pending/Committed/TimedOut）、maximize target id、`lifecycle_overlays_rendered`、closing/minimize/restore 条目及 progress
   - `render()` 与观测共用 `lifecycle_overlays_are_rendered()`，避免平行状态机

2. **真实 Mapped serial fixture**
   - `Mapped::test_maximize_commit_state()`（`#[cfg(test)]`）暴露 `is_maximized` / `is_pending_maximized` / `uncommitted_maximized` serials
   - 测试：`real_mapped_serial_distinguishes_configure_ack_and_commit`
   - 证明：configure 入队 → ack 不应用 → commit 经 `on_commit` 应用（**禁止**用 layout mock 空 `on_commit` 回答 F02）

3. **多输出 / scale / transform / source layer helpers**
   - `Fixture::add_output_with_scale_transform`
   - `Fixture::map_layer` / `unmap_layer` / `remap_layer`
   - 覆盖双屏异分辨率、1.25/2.0 scale、90° transform、layer map/unmap/remap 与 anchor 清理/恢复

4. **Opt-in 诊断计数**（默认关闭，无逐帧日志）
   - `NIRI_LIFECYCLE_DIAG=1` 或测试 `lifecycle_diag::enable()`
   - 计数：`queue_redraw_all`、`queue_redraw`、Genie create、snapshot variant/peak bytes、Tahoe region request/commit/capture
   - frame p50/p95/p99：既有 `NIRI_FRAME_TELEMETRY`；本任务补全 **frame_p50** 上报字段

5. **F01 现状可观测（不修复）**
   - `maximize_ongoing_plus_active_minimize_overlay_reports_suppressed_decision`
   - 构造：maximize transition ongoing + 非目标 Genie minimize active
   - 断言：overlay **存在且 active**，但 `rendered == false` 且 progress 仍可推进（时钟不暂停）
