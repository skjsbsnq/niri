- Per-screen DynamicIsland service

- Second ownership model or second notification queue

## Implementation

- `DynamicIslandOwnership.js`: resolve/sanitize pins + `screenPresentationRole`
- Pin `eventOwnerOutput` on first `showTransient` only (in-lease refresh keeps pin)
- `sessionOwnerOutput` on chip click/hover/expand/swipe with screenName
- Abort paths clear event pin (`clearEventOwner` + DND/timer)
- Owner `resting_media` is activity; non-owner always base clock
- Swipe collapse clears session; auto-expand claims session
- `computeTargetScreenName` → session > event > focused > first
- TopBar: `showTopbarTimeFallback: !enabled || !hideTopbarTime`
