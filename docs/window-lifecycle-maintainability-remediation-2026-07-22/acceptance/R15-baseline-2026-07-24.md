## 2. 锁定门槛（Go 条件）

### 2.1 R16（Genie 每帧分配）

- **默认 Go**（路线图：源码已确定存在 per-frame `Rc` / `HashMap` / `String` / `ShaderRenderElement::new`）。
- 本任务必须留下 **前测对照**：create 峰值字节、`render_genie` 每帧分配组数、理论 1s@60Hz 动画分配组次数。
- R16 实施后对照要求：`render_genie` 路径 `Rc::new` / `HashMap::from` / `String::from` / 逐帧 `ShaderRenderElement::new` 为零。

### 2.2 R17（lifecycle / foreign / glass 定向 redraw）

**Go 当且仅当同时满足：**

1. **Locatable 事件仍走 `queue_redraw_all`：** 目标 cluster 内至少一类可定位事件（foreign minimize / restore / maximize）在 headless 双输出下 `queue_redraw_all ≥ 1` 且主路径非 targeted `queue_redraw`。
2. **多输出浪费可测：** 双输出下 `queue_redraw_all` 将 **全部** output 标为 Queued；在 home_outputs=1 时 `waste_ratio = (outputs_queued−1)/outputs_total ≥ 0.5`。
3. **事件比率：** 成功 lifecycle 变更上 `queue_redraw_all` 次数 / 变更次数 ≥ 1.0（允许 roundtrip 次生计数 >1，但不得为 0）。

**单独静态 `rg` 命中数不构成 Go。** Glass 若 R14 已对 mapped root 定向，则 glass 子簇可记为已满足定向、不作为 R17 再实施对象；foreign/lifecycle 仍可独立 Go。

**No-go：** fallback 已极少、事件本质全局、或浪费比率 < 0.5 且无法证明 multi-output frame submission 成本。

### 2.3 R18（snapshot variant cache）

**Go 当且仅当同时满足：**

1. 在 block-out 路径上测得 `multi_peak / single_peak ≥ 1.5`；
2. **绝对峰值显著：** 实测 multi-variant peak ≥ **8 MiB**（1080p-class 量级）**或** 观测到 allocation stall / OOM fallback；
3. 默认生产 minimize 路径（无特设 window-rule）也出现多 variant，**或** 证明 Shell/screencast 常态走 multi-variant。

**No-go：** 仅理论表很大但默认单 variant、绝对值小、无 stall/OOM——不引入 cache 抽象。

### 2.4 R19（Tahoe render batching）

**Go 当且仅当同时满足：**

1. 结构上 N region → N capture（`note_tahoe_region_capture` 每 `render_region` 一次）且 N≥7 时 capture/region ≥ 0.9；
2. **GPU 主瓶颈：** tracy 或等价 GPU 计时证明 capture/blur 占 glass GPU 主成本（非仅 CPU request 计数）；
3. 可安全合并候选：≥2 region 共享 sample plane / kernel / xray / stacking epoch，且 union 面积增长 ≤ 1.2× 各 area 之和。

**No-go：** 无 GPU 计时、或 capture/blur 非主瓶颈、或无可安全合批几何。

### 2.5 指标分离规则

不得把 GPU、CPU、显存、协议流量混成单一“流畅度”结论。每条 go/no-go 只引用其门槛维度的原始样本。

## 7. Go / No-go 判定（对照第 2 节）

| 任务 | 判定 | 依据（对照锁定门槛） |
| --- | --- | --- |
| **R16** | **GO** | 源码确定 per-frame 4 组分配；前测 peak≈2.0 MiB（~1080 客户端）、240 alloc-groups/s@60Hz 对照已记录 |
| **R17** | **GO**（foreign/lifecycle） | (1) minimize/restore/maximize 均 `queue_redraw_all≥1`；(2) waste_ratio=0.50≥0.5；(3) 每变更 ≥1 redraw-all。**Glass mapped 已 targeted（R14）**，R17 实施范围应收窄到 lifecycle/foreign cluster，不重做 glass handler |
| **R18** | **NO-GO** | (1) multiplier=2.0 满足，但 (2) 实测 multi peak 462 KiB ≪ 8 MiB 且无 stall/OOM，(3) 默认路径单 variant。不引入 variant cache 抽象；理论 4K 表保留备查 |
| **R19** | **NO-GO** | 仅有 per-region capture 结构证据；缺 GPU 计时证明 capture/blur 为主瓶颈；无安全合批几何样本。不引入 batch owner |

## 8. 与 R00 基线关系

- R00 建立仪器；本任务在 **ownership 收敛后**复测，避免用重构前热点指导 R16–R19。
- `queue_redraw_all` 生产点 188→182：正确性主线略降，但 foreign lifecycle 仍全量 redraw-all。
- Genie 每帧分配 **未** 因 R01–R14 消失（符合预期；属 R16）。
- Glass redraw owner 已在 R14 收敛；R15 确认 mapped=targeted / unlocatable=fallback。
